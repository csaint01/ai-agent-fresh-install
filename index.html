Introduction

What Are We Building?

OpenClaw is an open-source AI agent that runs on your computer and connects to Telegram. Unlike a regular chatbot it can actually do things — write code, manage files, answer questions, automate tasks. You message it on Telegram like texting a friend and it responds using Claude AI.

🦞 OpenClaw

An open-source AI agent that connects Claude AI to your computer so it takes real actions — not just answers questions. Think of it as a personal assistant that never sleeps, never charges by the hour, and responds instantly.

🖥️ VirtualBox

A free program that creates a "virtual computer" inside your real computer. We run OpenClaw inside it for safety — if anything ever goes wrong you just delete the virtual machine. Your real files are never touched.

💡
Why use a Virtual Machine? OpenClaw can run commands and access files — that's what makes it powerful. But running it inside VirtualBox creates a safety barrier between the AI and your real computer. It's like letting a contractor work in your house but keeping them in one room.

Step 01

Create Your Virtual Machine

We create a virtual Ubuntu computer inside VirtualBox — like setting up a brand new computer inside a window on your existing screen.

  • 1
    Download and install VirtualBox
    Go to virtualbox.org, click Downloads, choose your operating system (Windows or Mac), and install it like any regular program. Accept all defaults.
  • 2
    Open VirtualBox → Click "New"
    Name it OpenClaw-Ubuntu. Under Type select Linux. Under Version select Ubuntu 64-bit. Click Next.
  • 3
    Set RAM to at least 6144 MB (6GB)
    Drag the memory slider to 6144 MB minimum. If your computer has 16GB+ RAM set it to 8192 MB (8GB). Too little RAM causes freezes.
  • 4
    Create a virtual hard disk — 80GB
    Choose Create a virtual hard disk now. Select VDI format, Dynamically allocated, and set size to 80 GB. Do not go smaller — 25GB will run out mid-install.
  • 5
    Set CPUs to 2
    In Settings → System → Processor, set the number of CPUs to 2. This makes the installation noticeably faster.
  • 6
    Attach the Ubuntu ISO file
    Settings → Storage → click the empty CD icon → choose the Ubuntu .iso file you downloaded. This is like inserting an installation disk.
  • 7
    Start the VM and install Ubuntu
    Click Start → choose "Install Ubuntu" → follow the prompts. Select your language, choose "Erase disk and install" (this only affects the virtual disk, not your real computer), set a username and password. Installation takes 10-20 minutes.
💡
After installation it asks to restart Click Restart Now. When it says "Remove the installation medium" just press Enter. The VM will boot into your fresh Ubuntu desktop. If a "Software Updater" popup appears click Install Now — these are important system updates.
Step 02

Set Up Ubuntu

Open the Terminal app (press Ctrl+Alt+T inside the VM) and run these commands. The terminal is a text-based control panel — you type commands and press Enter to run them.

💡
What is sudo? Many commands start with sudo — this means "run as administrator." Linux will ask for your password the first time. Type it and press Enter (the cursor won't move while typing — that's normal and intentional for security).
Ubuntu Terminal — type these and press Enter after each
# Update Ubuntu's list of available software
# && means "then run the next command if this one succeeds"
# -y means "yes to all prompts automatically"
sudo apt update && sudo apt upgrade -y

# Install curl (downloads files) and git (downloads code)
# We'll need both in the next steps
sudo apt install curl git -y
Step 03

Install Node.js

OpenClaw is built with Node.js — a programming environment that runs JavaScript outside of web browsers. We install it using NVM (Node Version Manager) which lets you easily switch versions if needed.

Ubuntu Terminal
# Download and run the NVM installer
# curl downloads the installer, | bash runs it immediately
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Reload your terminal profile so the nvm command is recognized
# source = "read and apply this file right now"
source ~/.bashrc

# Install Node.js version 22 (the version OpenClaw requires)
nvm install 22
nvm use 22

# Confirm it installed correctly — should show v22.x.x
node -v
v22.22.1
Step 04

Install OpenClaw

Install OpenClaw using npm — Node's built-in package manager. Think of npm like an app store for developer tools. The -g flag installs it globally so you can use it from any folder.

Ubuntu Terminal
# Install OpenClaw globally on your system
# This takes 1-3 minutes — lots of scrolling text is normal
npm install -g openclaw

# When finished you'll see: added XXX packages
# That means success — move to the next step
Step 05

Create Your Telegram Bot & Run Setup

Before running the OpenClaw setup wizard you need a Telegram bot token. This takes about 2 minutes and is how you'll communicate with your AI agent.

Part A — Get Your Telegram Bot Token

  • 1
    Open Telegram → Search for @BotFather
    BotFather is Telegram's official bot creator. Search for it and look for the blue ✅ verified checkmark — there are fakes without the checkmark. Tap Start or send any message to begin.
  • 2
    Send /newbot
    Type /newbot and send it. BotFather will ask for a display name (anything you want, e.g. "My AI Assistant") and then a username (must end in "bot", e.g. myassistant_bot).
  • 3
    Copy your bot token
    BotFather replies with a token that looks like 8546335464:AAHxxxxxxx... — copy the entire thing. This is your bot's password. Keep it private.
  • 4
    Get your Telegram User ID
    Search for @userinfobot on Telegram and send it any message. It instantly replies with your numeric User ID like 7326327152. Write this down — you'll need it shortly.

Part C — Set Your Allowlist (Important!)

This tells OpenClaw that only YOU are allowed to send it messages. Without this anyone who finds your bot can talk to it.

Ubuntu Terminal — replace YOUR_USER_ID with your number from @userinfobot
# Only allow your Telegram user ID to message the bot
# Replace 1234567890 with your actual user ID
openclaw config set channels.telegram.allowFrom '[1234567890]'
openclaw config set channels.telegram.groupPolicy allowlist
Step 06

Choose Your AI Model

OpenClaw defaults to Claude Sonnet — a powerful but more expensive model. For personal daily use Claude Haiku is cheaper and still very capable. You can switch anytime.

Claude Haiku — Recommended

Fast, affordable, capable. Handles coding, writing, file tasks, and general questions with ease. Perfect for personal use.

~$1 per million tokens input

Claude Sonnet — Default

More powerful for complex reasoning, deep coding, and analysis. Use when Haiku isn't giving good enough results.

~$3 per million tokens input

Ubuntu Terminal — choose one

openclaw models set claude-haiku-4-5-20251001

# Or keep Sonnet if you want more power
# openclaw models set claude-sonnet-4-6

Step 07

Make It Start Automatically on Boot

The final step — register OpenClaw as a system service so Ubuntu starts it automatically every time you boot up. After this you never need to manually start anything.

Ubuntu Terminal — run in order
# Register OpenClaw as a system service
# enable = start this automatically on every boot
systemctl --user enable openclaw-gateway.service

# Start it right now without rebooting
systemctl --user start openclaw-gateway.service

# Verify it's running
# Look for: Active: active (running)
systemctl --user status openclaw-gateway.service

# Confirm Telegram is connected
# Look for: Telegram | ON | OK
openclaw channels status
🎉
You're done! Open Telegram and message your bot. Find your bot by the username you created (e.g. @myassistant_bot) and send "Hello!" — it should respond within seconds. From now on it starts automatically every time Ubuntu boots. No manual steps needed.
Security

Quick Security Hardening

These two extra commands lock down your setup. Takes 1 minute and prevents the most common security warnings.

Ubuntu Terminal
# Tell OpenClaw to trust only the local machine as a proxy
openclaw config set gateway.trustedProxies '["127.0.0.1"]'

# Restart to apply all changes
systemctl --user restart openclaw-gateway.service

# Run the security audit to confirm everything is clean
openclaw status
# You want to see: 0 critical · 0 warn

Troubleshooting

If Something Goes Wrong

Bot not responding on Telegram

Fix
openclaw channels status
# If it shows error, restart the service:
systemctl --user restart openclaw-gateway.service

API key error (401 Unauthorized)

Fix — re-add your API key
openclaw models auth paste-token --provider anthropic
# Paste your sk-ant-... key when prompted
systemctl --user restart openclaw-gateway.service

node or openclaw command not found

Fix — reload your profile
# Reload the terminal profile so commands are recognized
source ~/.bashrc
nvm use 22

Gateway already running error

Fix — stop and restart cleanly
openclaw gateway stop
kill $(pgrep -f openclaw-gateway) 2>/dev/null
systemctl --user start openclaw-gateway.service
